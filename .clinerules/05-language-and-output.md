# Language and Output Rules

The project owner may write prompts, requirements and clarification messages in Polish.

You must understand Polish instructions correctly, but all project files, source code, comments, documentation, commit-style summaries, README content, Memory Bank files, configuration descriptions, variable names, function names, class names, database field names and user-facing technical text inside the repository must be written in English.

## Rules

- User prompts can be in Polish.
- Repository content must stay in English.
- Source code must be written in English.
- Code comments must be written in English.
- Markdown documentation must be written in English.
- Memory Bank files must be written in English.
- README.md and AGENTS.md must be written in English.
- Do not translate framework names, package names, API names, enum values or technical identifiers into Polish.
- If the user gives a Polish requirement, convert it into clear English project documentation before saving it.
- If clarification is needed, you may ask the user in Polish.
- Final explanations to the project owner may be in Polish unless they are saved into repository files.
