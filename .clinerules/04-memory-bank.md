# Memory Bank Rules

At the start of every task, read all files in memory-bank/.

Required files:
- projectbrief.md
- productContext.md
- activeContext.md
- systemPatterns.md
- techContext.md
- progress.md

Update rules:
- Update activeContext.md after each meaningful development step.
- Update progress.md after completed tasks, failed attempts and known issues.
- Update systemPatterns.md after architecture decisions.
- Update techContext.md after stack, setup or dependency changes.
- Update productContext.md after product/UX/scope changes.
- Update projectbrief.md only when core project goals change.

Memory Bank must stay concise, current and useful for continuing work in a new Cline session.
