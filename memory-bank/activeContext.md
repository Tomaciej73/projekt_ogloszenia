# Active Context

## Current Phase
**Phase 8 - Provider Research and Auth Hardening** (in progress)

A fully working application is running: user registration/login with HttpOnly cookie auth backed by session-backed JWTs plus PBKDF2+SHA512 password hashing, account activation via expiring email links, DB-backed login lockout after repeated failures, password reset flow with SMTP-delivered 6-digit codes, listing draft CRUD stored in PostgreSQL via Prisma, publication job creation with BullMQ, and a web frontend dashboard. The web runtime proxies API routes so browser clients can use the same origin instead of hardcoded `localhost` API URLs; the API authorizes `/media-files/...` requests before streaming objects from a private MinIO bucket. Active sessions can now be remotely revoked across devices with a five-second warning banner before forced sign-out. Docker Compose includes PostgreSQL 18, Redis 8, MinIO, and API/web/worker runtime containers running as the non-root `node` user.

## Active Decisions

- **Monorepo structure confirmed:** `apps/web`, `apps/api`, `apps/worker`, `packages/shared`, `packages/connectors`, `packages/config`
- **Tech stack confirmed:** pnpm workspaces, Node.js 24 LTS, PostgreSQL 18, Prisma v7 (with pg adapter), Redis 8, BullMQ, MinIO/S3
- **Runtime servers:** Plain Node.js HTTP servers (`apps/api/db-server.js`, `apps/web/front-server.js`) instead of NestJS/Next.js builds - chosen for immediate development velocity. NestJS source code exists as reference for future migration.
- **Runtime config source of truth:** Active JS runtimes now import per-runtime validated config from `packages/config` through small `runtime-config.js` bridges, so API/web/worker no longer bypass schema validation with secret or `localhost` fallbacks.
- **Workspace scripts are now honest smoke checks:** Root `pnpm test` and package-level `test` / `lint` scripts now run real type/syntax checks, while app `dev` scripts point at the active Node.js runtimes instead of placeholder echoes.
- **Provider roadmap:** OLX (1st) -> Vinted Pro (2nd) -> Facebook Marketplace (3rd). All `research_required`.
- **Version policy:** Latest LTS for runtimes, latest stable for frameworks, explicit SemVer when pinning/documenting versions, no `latest` Docker tags, verify from official sources.
- **Documentation discipline:** Every meaningful change must be documented in the appropriate `memory-bank` file, and every version-related change must use and record explicit SemVer values or a deliberate SemVer range.
- **Pinned package manager:** pnpm@11.10.0
- **Workspace protocol:** `workspace:*` for inter-package dependencies
- **Security:** All credentials from `.env` via dotenv. No hardcoded secrets in source code. Passwords use versioned PBKDF2-HMAC-SHA512 with a 16-byte salt and 220,000 iterations; legacy 100,000-iteration hashes are rehashed after successful login. New passphrases must be at least 15 characters and are checked through HIBP k-anonymity before storage. Browser auth uses an HttpOnly same-site cookie containing a session-backed JWT (user ID, opaque session ID, session version), and mutating requests require a same-origin CSRF token (`/auth/csrf` + `X-CSRF-Token`). Users can view and end active sessions. Password reset revokes all existing sessions atomically. New accounts are inactive until activated by email link or by the forgot-password activation flow. Accounts lock after 5 failed login attempts and are unlocked only by completing the password reset flow. Image uploads now accept only validated JPG/PNG/GIF/WebP payloads after server-side MIME sniffing and structural checks; direct presigned uploads are disabled in the active runtime.
- **Auth abuse protection:** `/auth/*` now has configurable Redis-backed rate limits in the active API runtime, and forgot-password code re-sends are additionally throttled through the existing `passwordResetRequestedAt` timestamp stored in PostgreSQL.
- **Resource protection and audit trail:** Authenticated uploads and publication requests use Redis-backed per-user limits; listings, active publication jobs, and media storage use configured quotas under a PostgreSQL advisory transaction lock. The active API and worker write allowlisted, secret-free `AuditLog` records for security-relevant account, marketplace, listing, media, and publication actions.
- **Runtime version source of truth:** The user-visible app version now comes from the shared `@multiportal/config` package SemVer through `packages/config/app-version.js`, so API health/logs and static HTML footers stay aligned even inside pruned Docker runtime images that no longer contain the monorepo root layout. Current application version: `0.4.18`.
- **Cross-device session revocation UX:** The API exposes a minimal authenticated `/auth/session-state` heartbeat that returns reasoned `401` responses while clearing both auth-state cookies, and the web runtime injects a shared watcher that warns revoked sessions for five seconds before clearing browser state and redirecting to sign-in.
- **Public visitor counter:** Every HTML page now renders a shared lower-right unique-visitor counter. The API persists only an HMAC-SHA256 hash of the normalized client IP in PostgreSQL, so reloads from the same address do not increase the total and the UI never exposes raw IP data.
- **Marketplace credential boundary:** Browser-facing marketplace-account responses use an explicit safe DTO and Prisma field allowlist, never returning provider user identifiers, access tokens, refresh tokens, or token-expiry metadata. The temporary mock account-linking endpoint is development-only; production returns `501` until an official OAuth flow is implemented.
- **Request body limits:** The active API now enforces byte limits while streaming JSON bodies into memory: standard JSON requests are capped at 1 MB, and `/media/upload` has a larger request cap sized for the existing 10 MB decoded image limit plus base64 overhead.
- **Cross-origin hardening:** Auth JSON endpoints now emit CORS headers only for allowed web origins, invalid auth preflights receive `403 Origin not allowed`, and the `mp_csrf` cookie is now `HttpOnly`.
- **Media access hardening:** MinIO objects are private. API startup waits for MinIO and replaces any legacy public policy before it accepts traffic. The web runtime sends `/media-files/...` to the API, which requires authentication and listing ownership before streaming an object, keeps responses private/no-store, and rejects traversal or malformed encoded keys. Direct browser-to-Minio access is removed.
- **Production network boundary:** `docker-compose.prod.yml` is mandatory for VPS deployment; it exposes only web on loopback for a TLS reverse proxy and removes host access to PostgreSQL, Redis, MinIO, its console, and the API.
- **Container runtime hardening:** API/web/worker images now build from the locked pnpm workspace with an explicit `.dockerignore`, use multi-stage `pnpm deploy` runtimes, load validated config from compiled `@multiportal/config` output instead of `tsx`, use explicit Prisma config paths for generate/migrate, run as the non-root `node` user, and expose Docker healthchecks.
- **Dependency posture:** The controlled SemVer upgrade pass is now on `0.4.1`, and `pnpm audit --json` is clean (`0` info/low/moderate/high/critical findings) after verified version bumps plus workspace overrides for vulnerable transitives.
- **Security tooling isolation:** Pentest/scanner tooling must stay outside the application runtime stack. Use separate temporary containers or local installs for ZAP/Burp/Trivy/Semgrep/Gitleaks, and remove those tool images/caches after assessments instead of keeping them alongside the app services.
- **Mail deliverability constraint:** The SMTP relay currently accepts messages, and runtime config now points to `noreply@manager.multiportal.site`, but real inbox delivery to providers like Gmail/Onet still depends on that mailbox actually existing on the relay plus aligned SPF/DKIM/DMARC.
- **Owner's changes are authoritative:** Port numbers, configuration values, file names chosen by the project owner must not be reverted or "corrected" by AI. See `.clinerules/01-project.md`.
- **Current local port assignments (from `.env`):** PostgreSQL 5243, Redis 6739, MinIO API 9000, MinIO Console 9001, API 3001, Web 3000. The production override publishes only web on `127.0.0.1:${WEB_PORT}`.
